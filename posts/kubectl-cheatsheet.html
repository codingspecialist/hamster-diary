kubectl 명령어 완벽 정리

2026-04-08 Kubernetes
kubectl은 Kubernetes 클러스터를 관리하기 위한 CLI 도구입니다. 이 글에서는 실무에서 자주 사용하는 명령어를 카테고리별로 정리합니다.

1. 클러스터 정보 & 컨텍스트 관리

명령어설명
kubectl cluster-info클러스터 정보 확인
kubectl version클라이언트/서버 버전 확인
kubectl config viewkubeconfig 설정 확인
kubectl config get-contexts모든 컨텍스트 목록
kubectl config current-context현재 활성 컨텍스트
kubectl config use-context <name>컨텍스트 전환
kubectl config set-context --current --namespace=<ns>기본 네임스페이스 변경

2. 네임스페이스 (Namespace)

명령어설명
kubectl get namespaces네임스페이스 목록
kubectl create namespace <name>네임스페이스 생성
kubectl delete namespace <name>네임스페이스 삭제
kubectl get all -n <namespace>특정 네임스페이스의 모든 리소스
kubectl get all --all-namespaces모든 네임스페이스의 리소스

3. Pod 관리

조회

명령어설명
kubectl get podsPod 목록 (현재 네임스페이스)
kubectl get pods -o wide상세 정보 (노드, IP 포함)
kubectl get pods --show-labels레이블 포함 조회
kubectl get pods -l app=nginx레이블 셀렉터로 필터링
kubectl get pods --field-selector status.phase=Running필드 셀렉터로 필터링
kubectl get pods -w실시간 변경 감시 (watch)

상세 정보 & 생성

명령어설명
kubectl describe pod <name>Pod 상세 정보
kubectl run <name> --image=<image>Pod 빠른 생성
kubectl run <name> --image=<image> --dry-run=client -o yamlYAML 생성 (적용 안 함)
kubectl delete pod <name>Pod 삭제
kubectl delete pod <name> --grace-period=0 --force강제 삭제

Pod 내부 접근

# 컨테이너 쉘 접속
kubectl exec -it <pod-name> -- /bin/bash

# 특정 컨테이너 접속 (멀티 컨테이너 Pod)
kubectl exec -it <pod-name> -c <container-name> -- /bin/sh

# Pod에서 명령어 실행
kubectl exec <pod-name> -- ls /app

# 파일 복사 (로컬 → Pod)
kubectl cp <local-path> <pod-name>:<remote-path>

# 파일 복사 (Pod → 로컬)
kubectl cp <pod-name>:<remote-path> <local-path>

4. Deployment 관리

명령어설명
kubectl get deploymentsDeployment 목록
kubectl describe deployment <name>Deployment 상세 정보
kubectl create deployment <name> --image=<image>Deployment 생성
kubectl set image deployment/<name> <container>=<new-image>이미지 업데이트 (롤링 업데이트)
kubectl rollout status deployment/<name>롤아웃 상태 확인
kubectl rollout history deployment/<name>롤아웃 히스토리
kubectl rollout undo deployment/<name>이전 버전으로 롤백
kubectl rollout undo deployment/<name> --to-revision=<n>특정 리비전으로 롤백
kubectl delete deployment <name>Deployment 삭제

5. 스케일링

명령어설명
kubectl scale deployment/<name> --replicas=<n>레플리카 수 조정
kubectl autoscale deployment/<name> --min=2 --max=10 --cpu-percent=80HPA 설정
kubectl get hpaHPA 목록
kubectl delete hpa <name>HPA 삭제

6. Service & Ingress

명령어설명
kubectl get servicesService 목록
kubectl get svcService 목록 (약어)
kubectl describe svc <name>Service 상세 정보
kubectl expose deployment <name> --port=80 --target-port=8080 --type=ClusterIPService 생성
kubectl expose deployment <name> --port=80 --type=NodePortNodePort Service 생성
kubectl expose deployment <name> --port=80 --type=LoadBalancerLoadBalancer Service 생성
kubectl get ingressIngress 목록
kubectl describe ingress <name>Ingress 상세 정보
kubectl port-forward svc/<name> <local>:<remote>포트 포워딩

7. ConfigMap & Secret

ConfigMap

명령어설명
kubectl get configmapsConfigMap 목록
kubectl create configmap <name> --from-literal=key=value리터럴로 생성
kubectl create configmap <name> --from-file=<path>파일에서 생성
kubectl describe configmap <name>ConfigMap 상세
kubectl get configmap <name> -o yamlYAML로 출력

Secret

명령어설명
kubectl get secretsSecret 목록
kubectl create secret generic <name> --from-literal=key=value리터럴로 생성
kubectl create secret docker-registry <name> --docker-server=... --docker-username=... --docker-password=...Docker 레지스트리 시크릿
kubectl get secret <name> -o jsonpath='{.data.key}' | base64 -dSecret 값 디코딩

8. 로그 & 디버깅

명령어설명
kubectl logs <pod-name>Pod 로그
kubectl logs <pod-name> -f실시간 로그 (follow)
kubectl logs <pod-name> --tail=100최근 100줄
kubectl logs <pod-name> --since=1h최근 1시간 로그
kubectl logs <pod-name> -c <container>특정 컨테이너 로그
kubectl logs <pod-name> --previous이전(재시작 전) 로그
kubectl logs -l app=nginx레이블로 여러 Pod 로그

디버깅 명령어

# 리소스 이벤트 확인
kubectl get events --sort-by=.metadata.creationTimestamp

# 노드 상태 확인
kubectl get nodes
kubectl describe node <node-name>
kubectl top nodes

# Pod 리소스 사용량
kubectl top pods
kubectl top pods --sort-by=memory

# 임시 디버그 컨테이너 (Ephemeral Container)
kubectl debug -it <pod-name> --image=busybox

# DNS 디버깅
kubectl run dns-test --image=busybox --rm -it -- nslookup <service-name>

9. YAML 적용 & 관리

명령어설명
kubectl apply -f <file.yaml>리소스 생성/업데이트
kubectl apply -f <directory/>디렉토리 내 모든 YAML 적용
kubectl apply -f <url>URL에서 YAML 적용
kubectl delete -f <file.yaml>YAML에 정의된 리소스 삭제
kubectl diff -f <file.yaml>적용 전 변경사항 확인
kubectl get <resource> <name> -o yaml리소스를 YAML로 출력
kubectl get <resource> <name> -o json리소스를 JSON으로 출력
kubectl edit <resource> <name>리소스 직접 편집

10. 레이블 & 어노테이션

명령어설명
kubectl label pod <name> env=prod레이블 추가
kubectl label pod <name> env=prod --overwrite레이블 덮어쓰기
kubectl label pod <name> env-레이블 삭제
kubectl annotate pod <name> description="my pod"어노테이션 추가
kubectl annotate pod <name> description-어노테이션 삭제

12. 유용한 팁

출력 포맷

# JSON 경로로 특정 필드 추출
kubectl get pods -o jsonpath='{.items[*].metadata.name}'

# 커스텀 컬럼
kubectl get pods -o custom-columns=NAME:.metadata.name,STATUS:.status.phase,NODE:.spec.nodeName

# 정렬
kubectl get pods --sort-by=.metadata.creationTimestamp

자동완성 설정 (Bash)

# bash 자동완성 활성화
source <(kubectl completion bash)

# 영구 설정
echo 'source <(kubectl completion bash)' >> ~/.bashrc

# 별칭 설정
alias k=kubectl
complete -o default -F __start_kubectl k

자동완성 설정 (Zsh)

# zsh 자동완성 활성화
source <(kubectl completion zsh)

# 영구 설정
echo 'source <(kubectl completion zsh)' >> ~/.zshrc

dry-run으로 YAML 생성

# Deployment YAML 생성
kubectl create deployment nginx --image=nginx --dry-run=client -o yaml > nginx-deploy.yaml

# Service YAML 생성
kubectl expose deployment nginx --port=80 --dry-run=client -o yaml > nginx-svc.yaml

# Job YAML 생성
kubectl create job my-job --image=busybox --dry-run=client -o yaml -- echo "hello"
