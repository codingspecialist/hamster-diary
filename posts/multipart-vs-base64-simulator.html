📖 Multipart vs JSON+Base64 비교 정리
📦 Multipart Form-Data — Content-Type: multipart/form-data; boundary=----
HTTP body를 boundary 문자열로 파트를 나눠서 각각의 필드와 파일을 원본 바이너리 그대로 전송합니다. 전통적인 <form enctype="multipart/form-data"> 방식이며, 파일 업로드의 표준입니다.
📋 JSON + Base64 — Content-Type: application/json
모든 데이터를 하나의 JSON 객체에 담고, 파일은 Base64로 인코딩하여 문자열 필드로 포함시킵니다. API 설계가 깔끔하고, 중첩 구조나 배열 등 복잡한 데이터를 표현하기 좋습니다.

핵심 차이점:

항목 Multipart JSON+Base64
전송 크기 원본 그대로 (작음) ~33% 증가
데이터 구조 평면적 (key-value) 중첩/배열 자유로움
서버 파싱 @RequestPart / MultipartFile @RequestBody + Base64 디코딩
스트리밍 가능 (대용량 OK) 전체 메모리 로드
브라우저 기본지원 FormData API FileReader 필요
적합한 상황 대용량 파일, 이미지 업로드 소규모 파일, REST API 통일
